Artificial Vision, using Vision Command

I want to use Vision Command as a NXT sensor to detect motion. Using VC as a motion detector and Ultrasonic sensor to detect distance, you could make a bot that it knows better the world where he runs.

At present, I am trying .NET platform with C# to program NXT Brick. I use the NXT# libraries developed by Bram Fokke. NXT# is a good work, but Ultrasonic Sensor response, must improve, :D.

Motion detector NXT Sensor is based in the web document written by Andrew Kirillov

I am researching the code to refine the class that I will share with NXT community.

Before finding Andrew Kirillov's work, I tried with:

  • CamCom
  • Legocam

CamCom
CamCom is an ActiveX.exe and makes in a VB-project classes available, which make the detection of color and movement possible. For this purpose detection areas must be defined. This can take place at run-time by program control or over the control-unit surface. Area definitions can be stored as configuration files, which can be generated and loaded also at run-time. The class makes events available, which the detection of movement or color indicates

Legocam
Wauters Kris developed a ActiveX control for use with the Vision Command Center from the Lego company. This software is provided as EmailWare, so you can copy and distribute this ActiveX to all your friends, as long as the persons that are using this software send an email to the author. wauters.kris@legomindstorms-belgium.org

Legocam documentation

B.van Kesteren developed an example to solve Tic tac toe game using legocam. Download Legocam Example
